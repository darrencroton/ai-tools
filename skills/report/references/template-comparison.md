# Comparison Template

Use for comparing implementations, architectures, branches, libraries, tools, or behavioural differences.

## Required Additional Sections

- `Comparison Criteria`
- `Options`
- `Tradeoffs`
- `Decision`
- `Decision Rationale`

## Completion Gate

Do not finish until the report contains:

- explicit comparison criteria
- enough evidence for each option
- a decision or a justified "no decision yet"
- rationale tied back to the criteria

## Template

```md
## Objective

## Scope

## Context

## Comparison Criteria

## Method

## Evidence

## Options

## Findings

## Tradeoffs

## Decision

## Decision Rationale

## Risks / Unknowns

## Recommended Next Actions

## Completion Status
```

## Notes

- Define the criteria before ranking options.
- If there are only two options, still use the plural `Options` section.
- If the evidence is incomplete, recommend the next discriminating check instead of forcing a weak conclusion.

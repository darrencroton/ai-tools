# Plan Template

Use for new features, implementation proposals, or scoped engineering plans.

## Required Additional Sections

- `Problem Statement`
- `Goals`
- `Non-Goals`
- `Constraints`
- `Options Considered`
- `Recommended Approach`
- `Implementation Phases`
- `Validation Plan`
- `Open Questions`

## Completion Gate

Do not finish until the report contains:

- a clear problem statement
- explicit goals and non-goals
- constraints that materially affect the plan
- a recommended approach with rationale
- phased implementation steps
- a validation plan

## Template

```md
## Objective

## Scope

## Context

## Problem Statement

## Goals

## Non-Goals

## Constraints

## Method

## Evidence

## Options Considered

## Recommended Approach

## Implementation Phases

## Validation Plan

## Risks / Unknowns

## Open Questions

## Recommended Next Actions

## Completion Status
```

## Notes

- Define what success looks like before proposing implementation detail.
- Keep non-goals explicit to prevent scope creep.
- If only one option is viable, still explain why alternatives were rejected or not pursued.
